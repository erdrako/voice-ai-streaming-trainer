Flujo end-to-end

  1. Browser graba o envia texto.
  2. FastAPI recibe eventos por WebSocket.
  3. STT transcribe, LLM responde en streaming.
  4. TTS sintetiza segmentos y el cliente reproduce audio.

Stack tecnologico

  • FastAPI
  • WebSocket
  • HTTPX async
  • Ollama
  • faster-whisper
  • Piper TTS
  • Redis
  • SQLite
  • Docker Compose
  • Terraform + Cloudflare

Buenas practicas aplicadas

  • Configuracion tipada con `pydantic-settings`.
  • Dependency Injection desde un composition root.
  • Contratos para STT, LLM, TTS, persistence y messaging.
  • Health checks separados entre liveness y readiness.
  • Logging estructurado, retries simples y errores estables.
  • Tests de use case con fakes, sin depender de modelos pesados.

Guia de portfolio

Que comunica esta app

Arquitectura

Separa transporte, orquestacion, reglas puras e integraciones para poder cambiar proveedores sin reescribir el workflow central.

Escalabilidad conceptual

Redis y SQLite son simples, pero muestran eventos, telemetria y persistencia desacoplada. El mismo patron se puede migrar a servicios mayores sin tocar los contratos.

Operabilidad

Incluye Docker Compose, readiness checks y opciones de exposicion por Cloudflare para demos reales sin subir la inferencia a la nube.

Narrativa de entrevista

La app sirve para hablar de latencia, streaming, resiliencia, intercambio de providers, observabilidad y decisiones de diseño con criterio de produccion.
